jsHub is an open source project created to eliminate the problems of web page tagging overload and data duplication. We make tools to make web site tagging easy, fast and secure. We host open standards that eliminate data inconsistencies across vendors.


What?

jsHub is an open source project created to eliminate the problems of web page tagging overload and data inconsistency.

The jsHub tag is written in JavaScript and distributed under the BSD license. It comes with a complete suite of automated tests and cross browser continuous integration testing to ensure its quality.

Based on standards

To ensure that data is only collected once the jsHub project is proposing a page markup standard based on HTML microformats, and released into the public domain, for common page metadata.

Comes with tools

We provide a Tag Inspector which shows all the data that is being collected on the page. No more debugging your implementation by monitoring HTTP traffic and unescaping query strings in your head!

Comes with best practice

There is a demo store which shows the tag, markup and tag inspector working together. Giving you a real life example of how to deploy the code.

Comes with tests

If you are a JavaScript developer, come and see how we do fully automated, cross-browser, continuous integration testing. How was our last build? You'll find the status and details of all the developer activity on the projects page.

Why?

It is common for web pages to contain many different tracking tags used for web analytics, advertising tracking, targetting, audience measurement, and more. Some sites have more than 10 tags per page.

Each tag adds either HTML or JavaScript to the page adding to its weight: more tags means larger pages which means longer load times.

And each tag ends up reading the same information from the page, such as its name, which can lead to inconsistencies. Problems associated with page loads may cause one tag to fire and another to misfire: that leads to inconsistent results and headaches for web site owners in interpreting data from different tools.

jsHub is designed to eliminate these problems by replacing many tags with a single tag. The single jsHub tag reads data once from the page, fires once, and replaces existing tags by sending data in native formats to different vendors’ products.

We believe that implementing a tag-based product on your site should not be an expensive exercise that locks you in to a single provider.

Who benefits from jsHub?

Vendors benefit because cross product data problems are eliminated as data is read once, they can use common code eliminating most of their tag maintenance, and tag installation becomes easier as jsHub becomes widely adopted.
See the standards pages for more.

Web site owners benefit from improved consistency of information across different products, lower tag weight and the assurance that the tag is widely tested and secure.
For more on the cost of implementation, see the enterprise page.

With the Tag Inspector web site owners can self-service and check tag implementation.
On the demo site, you can see the inspector in action.

End users benefit from faster page load times and greater transparency. With a single tag pages are smaller and faster to load. With the Tag Inspector an end user can quickly see what information is being gathered about them.
