  • It is common for web pages to contain many different tracking tags used for web analytics, ad tracking, targeting, audience measurement, and more. Some sites have more than 10 tags per page.

    Each tag adds either HTML or JavaScript to the page adding to its weight: more tags means larger pages which means longer load times. And each tag ends up reading the same information from the page, such as its name, which can lead to inconsistencies.

    A fresh approach

    jsHub is a suite of tools and a set of standards that allows you to declare data on your site once, and use it to power all the tag-based products you use.

  • Standard data

    To ensure that data is only declared once we propose a page markup standard based on HTML microformats, and released into the public domain, for common page metadata. 

            <div class="hpage">
          <span class="name">Homepage</span>
        </div>

    Declaring metadata means writing HTML, not JavaScript code.

    Microformat data can be indexed by search engines, and can be properly integrated with your CMS. And you only declare it once.

    We believe that implementing a tag-based product on your site should not be an expensive exercise in repeating yourself, and it should not lock you in to a single "universal tag" provider.

  • Implement tags for the last time

    If you run a site, jsHub can help you with consistent information across different products, lower tag weight and the assurance that the tag is widely tested and secure.

    For large sites, implementing a tab-based solution can be a significant expense. Using jsHub tags allows you to implement any further products simply by adding the right plugin to your tag.

    If you are maintaining a smaller site, you can use our Tag Inspector to self-service and check what is actually being sent from your tag implementation.

    Your end users benefit from faster page load times and greater transparency. With the Tag Inspector an end user can quickly see what information is being gathered on the page.

    → See how jsHub helps the enterprise

  • Use jsHub to power your product

    If your product uses tags to collect data from pages, then it is likely that your customers are spending time to reconcile your numbers with other products.

    It's also likely that implementing the tags is a significant expense. This makes it harder for prospective clients to try your product, and can cause a never-ending implementation.

    Unifying the markup on the site makes life easier for your customers and alleviates these problems.

    Cross product data problems are eliminated as data is read once. Your customers can use common code to remove most of their tag maintenance, and tag installation becomes easier as jsHub becomes widely adopted.
