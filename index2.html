  • It is common for web pages to contain many different tracking tags used for web analytics, audience measurement, optimization, and more. Some sites have more than 10 tags per page.

    Each tag adds some code to a web page, increasing its weight and causing longer load times. And each tag reads the same information from the page, which can lead to inconsistencies.

    A fresh approach

    jsHub is a suite of tools and a set of standards. It allows you to declare data once, and use it to power all the tag-based products on your site.

    → See it in action in the demo site

  • Implement tags for the last time

    For large sites, implementing a tag-based solution can be a significant expense. But with jsHub, it is only necessary to do it once. Any future products can be deployed simply by adding the relevant plugin to your tag.

    If you are maintaining a smaller site yourself, you can self-service, without writing JavaScript code. Use our friendly Tag Inspector to check what is actually being sent from your tags.

    Whatever the site, jsHub can help you with consistent information across different products, lower tag weight and the assurance that the tag is widely tested and secure.

    Your end users benefit from faster page load times and greater transparency. With the Tag Inspector an end user can quickly see what information is being gathered on the page.

  • Standard data

    To ensure that data is only declared once we propose a page markup standard based on HTML microformats, and released into the public domain, for common page metadata. 

            <div class="hpage">
          <span class="name">Homepage</span>
        </div>

    Declaring metadata means writing HTML, not JavaScript code.

    Microformat data can be indexed by search engines, and can be properly integrated with your CMS. And you only declare it once.

    We believe that implementing a tag-based product on your site should not be an expensive exercise in repeating yourself. It should not lock you in to a single "universal tag" provider.

  • Use jsHub to power your product

    If your product uses tags to collect data from pages, then it is likely that your customers are spending time to reconcile your numbers with other products.

    It's also likely that implementing the tags is expensive and time consuming. This makes it harder for prospective clients to try your product, and can cause a never-ending implementation.

    A standards-based implementation will make life easier for your customers and alleviate these problems.

    Cross product data problems are eliminated as data is read once. Your customers can use common code to remove most of their tag maintenance, and tag installation becomes easier as jsHub becomes widely adopted.
